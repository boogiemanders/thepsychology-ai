# Inzinna Extensions — Pricing Strategy

## Core Philosophy: Value-Based Pricing

Charge a percentage of the measurable value created, not a flat fee.
This makes pricing easy to sell ("you save $X, we charge $Y") and ties renewals to ROI.

---

## The Math

| Variable | Example |
|---|---|
| Clinicians in practice | 5 |
| Time saved per clinician per week | 2 hours |
| Total time saved per month | 40 hours |
| Clinician billing rate | $250/hour |
| **Monthly value created** | **$10,000** |

**Target charge: 10–15% of measured value**
- 10% → $1,000/month for the practice (~$200/clinician)
- 15% → $1,500/month for the practice (~$300/clinician)

## Pricing Tiers

### Solo Practitioner
- **$39/month** or **$390/year** (~$32.50/mo, 2 months free)
- One clinician, all features (both extensions)

### Small Group (2–5 clinicians)
- **$149/month** or **$1,490/year**
- Shared license, all clinicians on one account

### Mid-Size Practice (6–10 clinicians)
- **$299/month** or **$2,990/year**

### Enterprise (11+ clinicians)
- Contact for pricing
- Custom onboarding, priority support, custom VOB templates

## Add-On: Managed Service / Retainer

On top of the base subscription, offer a **custom development retainer** for
practice-specific needs (custom intake fields, new EHR integrations, etc.).

| Tier | Price | What's Included |
|---|---|---|
| Support | +$99/month | Direct Slack/email access, priority bug fixes, 48hr SLA |
| Dev Retainer | +$400–800/month | ~4–8 hrs/month of custom development at ~$100/hr |

**Rule of thumb:**
- Feedback that benefits all customers → fix for free, ship to everyone
- Custom work that only helps one practice → retainer work

## Trial Strategy

### Free 30-Day Team Trial
1. Practice installs both extensions for the full team
2. Extensions auto-track usage (action counts, estimated time saved)
3. At end of trial, review with practice: "Your team ran 87 actions last month.
   At 10 minutes saved per action, that's 14.5 hours / $3,600 of value."
4. Propose a monthly price at 10–15% of measured value
5. They see the ROI clearly → easy yes

### Usage Tracking (non-PHI)
Built into the popup — shows "Time saved this month" counter.
No patient data sent, just action counts + timestamps.

## Go-To-Market Channels

### Immediate (Highest Leverage)
1. **Sosa Partners** — your existing billing partner, already in default VOB config.
   They work with dozens of practices. One conversation = ready-made pipeline.

2. **SimplePractice community** — Facebook group (40k+ members), official forum.
   A post like "built a Chrome extension that auto-fills your intake notes" spreads organically.

3. **ZocDoc provider groups** — Facebook groups for ZocDoc providers.

### Medium Term
4. **Psychology Today / APA / state psych associations** — newsletters, vendor directories,
   APA Practice Central, state association email lists.

5. **LinkedIn content** — posts targeting therapists/psychologists about admin time savings.
   Mental health practitioners are active on LinkedIn.

### Growth
6. **Referral program** — "Refer a colleague → 1 month free" link built into the extension
   feedback panel. Word-of-mouth inside a practice → between practices.

7. **Chrome Web Store** — Once 3–5 paying customers validate the product,
   publish publicly with "HIPAA-safe, works with SimplePractice + ZocDoc."

## One-Time Setup: `extension_license_key` DB Column

Before the subscription gate goes live, run this in Supabase SQL editor:

```sql
ALTER TABLE users ADD COLUMN IF NOT EXISTS extension_license_key TEXT UNIQUE;
```

Existing paying subscribers won't get a key automatically — they'll need to either:
- Trigger a new checkout (key generated by Stripe webhook), OR
- Have the key manually generated and emailed to them as a one-time migration step

To backfill existing pro users:
```sql
UPDATE users
SET extension_license_key = gen_random_uuid()::text
WHERE subscription_tier = 'pro' AND extension_license_key IS NULL;
```
