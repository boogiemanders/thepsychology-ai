# Content Folder

This folder is split by purpose so future-state strategy does not get mixed with shipped-product messaging or loose draft writing.

## Structure

- `strategy/` — planning docs for where the company may go next
- `research/` — competitive teardowns, pain-point research, and positioning inputs
- `founder-drafts/` — rough writing, idea dumps, and non-core content experiments

## Current Rule Of Thumb

If a doc describes a product or feature that does not exist yet, it belongs in `strategy/`.
If a doc gathers evidence, market observations, or competitor analysis, it belongs in `research/`.
If a doc is exploratory writing that should not drive product or homepage decisions by default, it belongs in `founder-drafts/`.
